> Part of [Tak_OS](https://github.com/tak0dan/Tak_OS) · parent: [`/etc/nixos/`](../README.md)

```
packages/
├── all-packages.nix       ← master hub — imports every group below
├── core.nix               ← essential CLI tools, base utilities
├── development.nix        ← compilers, debuggers, dev toolchains
├── browsers.nix           ← web browsers
├── communication.nix      ← chat, email, social clients
├── media.nix              ← audio/video players, image viewers
├── games.nix              ← native Linux games

├── kde.nix                ← KDE Plasma specific packages

└── disabled/
    └── disabled-packages.nix  ← packages kept for reference, not imported
```

## Description

`packages/` separates **what software is installed** from **how the system is
configured**. Every file here is a Nix expression that returns a list of
packages added to `environment.systemPackages` (or equivalent).

`all-packages.nix` is the single import point wired into `configuration.nix`:

```nix
imports = [
  ./packages/all-packages.nix
];
```

All other files are imported from there. To disable a category, comment out
its line in `all-packages.nix` — no other file needs to change.

### Intentional Duplication

Some packages appear in more than one group. This is **deliberate**:

- Each module must remain independently usable.
- Removing one group must never silently break another.
- Local dependency = visible dependency.

### nixorcist-managed packages

Packages installed through `nixorcist` land in:

```
../nixorcist/generated/all-packages.nix   ← separate hub
../nixorcist/generated/.modules/          ← one .nix per package
```

Do **not** mix nixorcist-generated files with hand-written files here.

## Adding a Package

**Manual (one-off):**

```bash
# Edit the appropriate category file
sudoedit /etc/nixos/packages/core.nix

# Rebuild
sudo nixos-rebuild switch
```

**Via nixorcist (recommended):**

```bash
sudo nixorcist install <package-name>
sudo nixorcist all
```

## Removing a Package

```bash
sudo nixorcist delete <package-name>
sudo nixorcist all
```

Or remove it from the relevant `.nix` file and rebuild.

## Adding a New Category

1. Create `packages/<category>.nix` with the package list.
2. Import it in `packages/all-packages.nix`.
3. Rebuild with `sudo nixos-rebuild switch`.

---

## Code of Conduct

- Keep each file focused on one logical category.
- Do not add system service configuration here — that belongs in `modules/`.
- Do not edit `autogenerated.nix` or any file under `../nixorcist/generated/` by hand.
- Use `disabled/disabled-packages.nix` to park packages you want to remember but not install.

---

## Upgrade Guide

```bash
# After a channel update, some package names may change.
# The smart rebuild script resolves these automatically:
sudo bash /etc/nixos/scripts/nix-rebuild-smart.sh

# Or let nixorcist handle it interactively:
sudo nixorcist rebuild
```
