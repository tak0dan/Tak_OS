## How Package Loading Works

`all-packages.nix` is the single import point into the module system.
`modules/system-packages.nix` assembles the final list and applies filtering.
The flow is:

```
packages/all-packages.nix
  imports every packages/*.nix group
    ↓
modules/system-packages.nix
  applies filterPkgs (disabled list + kool.disabledPackages option)
    ↓
environment.systemPackages
```

To disable an entire category, comment out its line in `all-packages.nix`.
No other file needs to change.

### Intentional duplication

Some packages appear in more than one group. This is **deliberate**:

- Each group must remain independently usable.
- Removing one group must never silently break another.
- A visible local dependency is better than a hidden shared one.

NixOS deduplicates packages at evaluation time, so there is no installation
overhead from listing the same package in two groups.

## The Disabled Packages System

Tak_OS provides two ways to exclude a package from the entire system without
editing every file that mentions it.

### 1. File-managed list

`packages/disabled/disabled-packages.nix` contains a plain Nix list of
package name strings:

```nix
[ "steam" "discord" "telegram-desktop" ]
```

Managed by the CLI tools:

```bash
nixos-comment   <pkg>    # adds pkg to the disabled list
nixos-uncomment <pkg>    # removes pkg from the disabled list
```

Use this for **permanent** exclusions tracked in version control.

### 2. Inline option: `kool.disabledPackages`

Declared in `configuration.nix` for quick one-off overrides:

```nix
kool.disabledPackages = [
  "discord"
  "telegram-desktop"
];
```

This is a NixOS option defined in `modules/system-packages.nix`. Use it for
temporary exclusions you have not yet committed to the file-managed list.

### How filtering is applied

Both lists are combined in `modules/system-packages.nix` into the `filterPkgs`
function passed to every module via `_module.args`:

```
filterPkgs (list)
  = removes packages whose name is in packages/disabled/disabled-packages.nix
  + removes packages whose name is in kool.disabledPackages
```

A package in either list is excluded everywhere. No need to edit individual
category files.

## Adding and Removing Packages

### Add a package manually

```bash
sudoedit /etc/nixos/packages/core.nix
sudo nixos-rebuild switch
```

### Add via nixorcist (reversible)

```bash
sudo nixorcist install <package-name>
sudo nixorcist all
```

nixorcist writes to `nixorcist/generated/`, not to `packages/`. Promote to
a category file manually when you want a permanent declaration.

### Remove a package

```bash
# Via CLI
sudo nixorcist delete <package-name>
sudo nixorcist all

# Or manually
sudoedit /etc/nixos/packages/core.nix
sudo nixos-rebuild switch
```

### Temporarily disable without removing

```bash
nixos-comment   <pkg>    # disable globally
nixos-uncomment <pkg>    # re-enable
```

### Add a new category

1. Create `packages/<category>.nix` with a package list.
2. Import it in `packages/all-packages.nix`.
3. Rebuild.

## Code of Conduct

1. **Keep each file focused on one logical category.**

2. **Do not add system service configuration here.**
   Service configuration belongs in `modules/`.

3. **Never edit `autogenerated.nix` or files in `nixorcist/generated/`.**
   Those files are machine-owned.

4. **Use `disabled/disabled-packages.nix` to park packages** you want to
   remember but not install.

5. **All category files must be importable independently** — no dependencies
   on variables or functions from sibling files.
